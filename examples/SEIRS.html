SEIRS Model example

The following example illustrates how to run the SEIRS Model using the Fair Data Pipeline with the pyDataPipeline and [pySimpleModel]() package.

config.yaml

The user should supply a config.yaml file describing a code run with the following inputs:

SEIRS_model/parameters

and the following outputs:

SEIRS_model/results/model_output/python
SEIRS_model/results/figure/python

an example config is provided within the pyDataPipeline package located in data_pipeline_api/ext/SEIRSconfig.yaml and contains the following:

 1run_metadata:
 2  default_input_namespace: rfield

 4  script: python3 data_pipeline_api/ext/SEIRSModelRun.py
 5  remote_repo: https://github.com/FAIRDataPipeline/pyDataPipeline.git

The script specified in run_metadata will be called by fair run

fair pull

Using the CLI tool, the command fair pull will indentify any data products listed in the register field of the config.yaml file and download them to the local data store as set by fair init and register the corresponding metadata to the local registry.

The following commands should be run after activating a poetry shell from the CLI tool:

fair init --ci
fair pull data_pipeline_api/ext/SEIRSconfig.yaml

The --ci flag can be used to use temporary directories. If using this flag the above code should be run from the local_repo directory

Once the fair pull command has been run the following data product should now be stored in the local registry

  1. SEIRS_model/parameters

fair run

In order to run the script specified in run_metadata['script'] field of the config.yaml the command fair run from the CLI Tools should be used, Once again if the --ci flag was used with the fair init command then this must be from the local_repo directory. This should also only be run after activating a poetry shell from the command line and installing the pyDataPipeline package within the poetry virtual environment following the Installation instructions.

fair run data_pipeline_api/ext/SEIRSconfig.yaml

This will call fair run which will produce a code run and the appropriate output using the script provided in the run_metadata field of the config.yaml file.

The following code will be executed when using the data_pipeline_api/ext/SEIRSconfig.yaml config file, which is located in the data_pipeline_api/ext/SEIRSModelRun.py Python script

 1import os
 2import platform
 3import sys
 4
 5sys.path.append(str(os.getcwd()))
 6import data_pipeline_api as pipeline  # noqa: E402
 7import simpleModel as simpleModel  # noqa: E402
 8
 9token = str(os.environ.get("FDP_LOCAL_TOKEN"))
10script = os.path.join(str(os.environ.get("FDP_CONFIG_DIR")), "script.sh")
11if platform.system() == "Windows":
12    script = os.path.join(str(os.environ.get("FDP_CONFIG_DIR")), "script.bat")
13config = os.path.join(str(os.environ.get("FDP_CONFIG_DIR")), "config.yaml")
14handle = pipeline.initialise(token, config, script)
15
16initial_parameters = pipeline.link_read(handle, "SEIRS_model/parameters")
17model_output = pipeline.link_write(
18    handle, "SEIRS_model/results/model_output/python"
19)
20model_plot = pipeline.link_write(handle, "SEIRS_model/results/figure/python")
21
22# Model Code
23states = {"s": 0.999, "e": 0.001, "i": 0, "r": 0}
24
25init_params = simpleModel.readInitialParameters(initial_parameters)
26
27alpha = init_params["alpha"]
28beta = init_params["beta"]
29inv_gamma = init_params["inv_gamma"]
30inv_omega = init_params["inv_omega"]
31inv_mu = init_params["inv_mu"]
32inv_sigma = init_params["inv_sigma"]
33R0 = init_params["R0"]
34
35sm = simpleModel.SEIRS_Model(
36    states, 1000, 5, alpha, beta, inv_gamma, inv_omega, inv_mu, inv_sigma
37)
38
39simpleModel.SEIRS_Plot(sm, model_plot)
40
41simpleModel.write_model_to_csv(sm, model_output)
42
43pipeline.finalise(token, handle)
